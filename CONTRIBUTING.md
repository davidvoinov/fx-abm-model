# Contributing

Спасибо за вклад в проект.

## Ветки

- `main` — стабильная ветка
- feature-ветки: `feature/<short-name>`
- bugfix-ветки: `fix/<short-name>`

## Pull Request checklist

Перед открытием PR:

1. Обновите ветку от актуального `main`
2. Убедитесь, что код запускается
3. Прогоните проверки:
   - `python test_theta.py`
   - `python test_full.py` (если изменение затрагивает симуляцию/метрики)
4. Добавьте описание в PR:
   - цель изменения,
   - как проверить,
   - потенциальные эффекты на метрики/графики.

## Код-стиль

- Сохраняйте существующий стиль проекта
- Не смешивайте рефакторинг и функциональные изменения в одном PR
- Держите PR компактными и предметными

## Issue

Для багов указывайте:

- шаги воспроизведения,
- ожидаемое и фактическое поведение,
- минимальные параметры симуляции,
- traceback/логи.
